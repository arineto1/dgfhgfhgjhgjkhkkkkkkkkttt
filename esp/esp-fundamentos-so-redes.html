2. Sistemas Operacionais: Processos e Memória

Núcleo e Modos de Operação

  • Modo Usuário: Restrito. Aplicações sem acesso direto ao hardware.
  • Modo Kernel: Acesso total. Transição ocorre via System Call (Trap).
  • Arquiteturas:
    • Monolítico: Tudo no kernel (Rápido/Instável). Ex: Linux, Windows.
    • Microkernel: Mínimo no kernel (Lento/Estável). Ex: Minix.

Gerenciamento de Processos

  • Processo vs. Thread: Processo é o programa em execução (isolado). Thread é "leve", compartilha memória mas tem sua própria Pilha (Stack) e PC.
  • Ciclo de Vida (Estados):
    • Running → Ready: Ocorre por Preempção (fim do quantum ou prioridade). O processo não escolhe sair.
    • Running → Waiting: Voluntário (I/O).
  • Deadlock (Condições de Coffman): Para haver impasse, 4 condições devem ser simultâneas: 1) Exclusão Mútua, 2) Posse e Espera, 3) Não Preempção, 4) Espera Circular.

Memória e Paginação

  • Paginação: Memória Física dividida em Quadros; Lógica em Páginas. Resolve fragmentação externa.
  • Thrashing: Sistema ocupa mais tempo trocando páginas (swap) do que processando. Causa: excesso de processos (multiprogramação alta).
  • TLB: Cache de hardware essencial para velocidade da tradução de endereços.

3. Linux: Comandos e Diretórios (Decoreba)

Permissões (chmod)

Memorize a tabela octal: R=4 (Leitura), W=2 (Escrita), X=1 (Execução).

Exemplo: chmod 751 arquivo
Dono: 4+2+1 = 7 (Total)
Grupo: 4+0+1 = 5 (Leitura/Execução)
Outros: 0+0+1 = 1 (Execução)

Principais Comandos e Flags

mv Mover ou Renomear.
cp -r Copiar diretórios (recursivo).
rm -rf Remove diretórios sem perguntar (Force).
tail -f Monitora logs em tempo real (append).
grep Filtra texto. Flags: -i (ignora case), -v (inverte busca).
ps aux / top Visualizar processos.
kill -9 Mata processo imediatamente (SIGKILL).

Estrutura de Diretórios (FHS)

  • /etc: Configurações (texto).
  • /dev: Dispositivos (hardware).
  • /bin vs /sbin: Usuário vs Admin (root).
  • /proc: Sistema de arquivos virtual (info kernel).

4. Redes: OSI, TCP/IP e Protocolos

Modelo OSI vs. TCP/IP

  • Física (1): Bits, Cabos, Hub.
  • Enlace (2): Quadros, MAC Address, Switch.
  • Rede (3): Pacotes, IP, Roteador (Melhor caminho).
  • Transporte (4): Segmentos, TCP/UDP (Portas).
  • Sessão, Apresentação, Aplicação: No TCP/IP, condensadas na camada de Aplicação.

Protocolos e Portas (Tabela Mestra)

Protocolo Porta Tipo Função Chave
SSH 22 TCP Acesso remoto seguro (Substitui Telnet).
DNS 53 UDP/TCP Traduz Nomes ↔ IP.
DHCP 67/68 UDP Distribui IPs (DORA).
HTTP 80 TCP Web (Texto claro).
HTTPS 443 TCP Web Segura (SSL/TLS).
POP3 110 TCP Recebe e baixa/deleta do servidor.
IMAP 143 TCP Recebe e sincroniza (mantém no servidor).
SMTP 25/587 TCP Envia e-mail (Push).
RDP 3389 TCP Acesso Remoto Windows.

TCP vs. UDP

  • TCP: Confiável, Orientado a Conexão (Handshake 3 vias), Ordenado, Lento. (Ex: Web, E-mail).
  • UDP: Não confiável, Sem conexão ("Fire and Forget"), Rápido. (Ex: Streaming, VoIP, DNS).

5. Endereçamento IP e Cálculos

IPv4 vs. IPv6

  • IPv4: 32 bits. Possui Broadcast. Fragmentação nos roteadores.
  • IPv6: 128 bits (Hexadecimal). Não possui Broadcast (usa Multicast/Anycast). Cabeçalho fixo.

Cálculo de Sub-rede (Ex: /26)

  1. Bits emprestados: 26 - 24 = 2 bits.
  2. Sub-redes: 2² = 4.
  3. Hosts totais: 32 - 26 = 6 bits sobram → 2⁶ = 64.
  4. Hosts Úteis: 64 - 2 = 62.
  5. Máscara: /26 = 255.255.255.192.

6. Windows Server e Segurança

  • Active Directory (AD): Baseado em LDAP e Kerberos.
  • GPO (Group Policy): Aplicação em ordem LSDOU (Local, Site, Domínio, OU). A última sobrescreve.
  • Segurança Wi-Fi:
    • WEP: Inseguro (RC4).
    • WPA2: Seguro (AES/CCMP).
    • WPA3: Atual (SAE, proteção contra dicionário).
