Módulo 1: A Base da Linguagem C

1.1 Tipos de Dados Primitivos e Limites

  • int: Inteiro (comum: 4 bytes).
  • char: Caractere (1 byte).
  • float: Ponto flutuante, precisão simples.
  • double: Ponto flutuante, precisão dupla.
  • void: Ausência de tipo/valor.
  • Modificadores:
    • signed (padrão) / unsigned: unsigned dobra o limite positivo, não permite negativos.
    • short / long: Alteram o tamanho/faixa de int e double.

Ponto de Atenção (Memorizar): Overflow

  • unsigned char uc = 255; uc++; → uc vira 0.
  • signed char sc = 127; sc++; → sc vira -128.

1.2 Operadores e Precedência (Decisivo em Provas)

A ordem de avaliação é a chave. Memorize a hierarquia:

  1. Pós-fixado/Acesso: (), [], ->, . , var++, var--
  2. Unário/Pré-fixado: ++var, --var, ! (NÃO), * (desref.), & (endereço), sizeof
  3. Multiplicativo: *, /, %
  4. Aditivo: +, -
  5. Relacional: >, <, >=, <=
  6. Igualdade: ==, !=
  7. Lógicos: && (E), || (OU)
  8. Ternário: ?:
  9. Atribuição: =, +=, -= etc.

Exemplo Clássico de Prova: int a=5, b=3, c=2; int res = ++a * b-- - c;

  1. ++a: a vira 6. Usa-se 6.
  2. b--: Usa-se 3. Depois, b vira 2.
  3. Expressão: 6 * 3 - 2
  4. Multiplicação primeiro: 18 - 2
  5. Resultado: res = 16.

1.3 Estruturas de Controle

  • Decisão:
    • if-else: Condicional simples ou aninhada.
    • switch-case: Múltipla escolha para inteiros/char. Atenção: Sem break, ocorre fall-through (continua para o próximo case).
  • Repetição:
    • while: Testa a condição antes. Pode não executar.
    • do-while: Testa a condição depois. Executa pelo menos uma vez.
    • for: Ideal para número definido de iterações (inicialização, condição, incremento).
  • Controle de Laço:
    • break: Sai do laço/switch imediatamente.
    • continue: Pula para a próxima iteração do laço.

1.4 Entrada e Saída (<stdio.h>)

  • printf("%d %f %c %s",...): Escreve formatado.
  • scanf("%d", &variavel): Lê formatado. Requer o & (endereço).

Módulo 2: Estruturando Dados e Código

2.1 Funções

  • Passagem por Valor (Padrão): A função recebe uma cópia. A variável original não é alterada.
  • Passagem por Referência (com Ponteiros): A função recebe o endereço (&). A variável original pode ser alterada usando *ponteiro.
void por_valor(int x) { x++; } // Não altera o original
void por_ref(int *x_ptr) { (*x_ptr)++; } // Altera o original
  • Recursividade: Função que chama a si mesma. Requer um caso base para parar. Ex: Fatorial.

2.2 Vetores (Arrays)

  • Coleção de itens do mesmo tipo em memória contígua. Índice começa em 0.
  • Relação com Ponteiros (Memorizar): O nome de um vetor é um ponteiro para seu primeiro elemento.
    • vetor é o mesmo que &vetor[0].
    • vetor[i] é o mesmo que *(vetor + i).

2.3 Strings (Vetores de char)

  • Não há tipo string. É um char array terminado com o caractere nulo \0.
  • Biblioteca <string.h>:
    • strlen(s): Comprimento (não conta \0).
    • strcpy(dest, orig): Copia string. Inseguro.
    • strcat(dest, orig): Concatena strings. Inseguro.
    • strcmp(s1, s2): Compara. Retorna 0 se iguais.

2.4 Structs

  • Agrupa variáveis de tipos diferentes sob um único nome. Acesso com o operador ponto (.).
struct Aluno { int mat; char nome; };
struct Aluno a1;
a1.mat = 123;

Módulo 3: Ponteiros e Memória Dinâmica

3.1 Conceitos Fundamentais de Ponteiros

  • Ponteiro: Variável que armazena um endereço de memória.
  • Operador & (Endereço de): Obtém o endereço de uma variável. int *p = &var;
  • Operador * (Desreferência): Acessa o valor no endereço apontado. *p = 10; (altera var).

Ponto de Atenção (Questão Clássica - [1]):

void func(int *a, int *b) { *a = *a + *b; }
int main() {
    int x=5, y=3;
    func(&x, &y); // x é alterado para 8
    printf("%d", x); // Imprime 8
}

3.2 Alocação Dinâmica de Memória (<stdlib.h>)

  • malloc(tamanho_em_bytes): Aloca memória no heap. Retorna void*. Memória contém "lixo".
  • calloc(qtd_itens, tamanho_item): Aloca e inicializa com zero.
  • free(ponteiro): Obrigatório! Libera a memória alocada para evitar memory leak.

Módulo 4: Manipulação de Arquivos (<stdio.h>)

  • Abertura: FILE *arq = fopen("nome.txt", "modo");
  • Fechamento: fclose(arq); (Sempre fechar!)
  • Modos Essenciais (Memorizar):
    • "r": Leitura (Read). Arquivo deve existir.
    • "w": Escrita (Write). Apaga o conteúdo se existir, ou cria.
    • "a": Anexar (Append). Escreve no fim, ou cria.
    • Adicionar + (ex: "r+") permite leitura e escrita.
    • Adicionar b (ex: "rb") para modo binário.
  • Funções:
    • Texto: fprintf(), fscanf(), fgets().
    • Binário: fwrite(), fread().

Módulo 5: Algoritmos de Pesquisa e Ordenação

Análise de Complexidade (Big O):

  • O(1): Constante
  • O(log n): Logarítmica (muito eficiente)
  • O(n): Linear
  • O(n log n): Linearítmica (ordenação eficiente)
  • O(n²): Quadrática (ordenação simples)

Pesquisa:

  • Sequencial: Percorre tudo. O(n). Não precisa de ordenação.
  • Binária: Divide pela metade a cada passo. O(log n). Requer dados ordenados.

Tabela Comparativa de Ordenação (Memorizar):

Algoritmo Complexidade Média Pior Caso Espaço Auxiliar Estável?
Bubble Sort O(n²) O(n²) O(1) Sim
Selection Sort O(n²) O(n²) O(1) Não
Insertion Sort O(n²) O(n²) O(1) Sim
Merge Sort O(n log n) O(n log n) O(n) Sim
Quick Sort O(n log n) O(n²) O(log n) Não

Módulo 6: Estruturas de Dados Fundamentais

  • Lista Encadeada: Nós com dado e *proximo. Inserção/remoção rápida, busca lenta (O(n)).
  • Pilha (Stack): LIFO (Last-In, First-Out). O último a entrar é o primeiro a sair.
    • Operações: push (insere), pop (remove).
  • Fila (Queue): FIFO (First-In, First-Out). O primeiro a entrar é o primeiro a sair.
    • Operações: enqueue (insere), dequeue (remove).
  • Árvore Binária de Busca (ABB):
    • Propriedade: Esquerda < Raiz < Direita.
    • Busca eficiente: O(log n) em média.
  • Percursos em Árvore (Questão Comum - [1]):
    • Pré-ordem: Raiz, Esquerda, Direita.
    • Em-ordem: Esquerda, Raiz, Direita (visita em ordem crescente na ABB).
    • Pós-ordem: Esquerda, Direita, Raiz.
