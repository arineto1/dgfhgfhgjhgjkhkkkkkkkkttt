1. Os Quatro Pilares da POO: Definições Essenciais

Pilar Palavra-Chave Definição Rápida e Foco de Concurso
Abstração Essencial Focar nas características e comportamentos relevantes de um objeto, ignorando detalhes. É o ato de filtrar a complexidade para criar um modelo útil.[1, 2, 3]
Encapsulamento Proteger Agrupar dados (atributos) e métodos em uma classe, ocultando a implementação interna. Garante a integridade dos dados através de acesso controlado (getters/setters).[4, 5]
Herança É-UM (IS-A) Uma classe (filha) herda atributos e métodos de outra (pai). Permite a reutilização de código, mas cria alto acoplamento.[4, 6, 7]
Polimorfismo Muitas Formas Um objeto pode ser tratado de múltiplas maneiras. Uma chamada de método (`animal.emitirSom()`) executa ações diferentes dependendo do tipo real do objeto (Cachorro, Gato).[8, 9]

2. Decisões Críticas de Design: Comparações para a Prova

Herança vs. Composição (Questão frequente de design)

Critério Herança Composição
Relação "É-UM" (IS-A). `Carro` é um `Veiculo`. "TEM-UM" (HAS-A). `Carro` tem um `Motor`.
Acoplamento Alto. Alterações na superclasse afetam as subclasses. Baixo. Interação via interface pública, mais flexível.
Flexibilidade Baixa. Definida em tempo de compilação (estática). Alta. Pode ser alterada em tempo de execução (dinâmica).
Princípio Chave Usar com cautela. "Prefira Composição sobre Herança".

Classe Abstrata vs. Interface (Diferença conceitual e prática)

Critério Classe Abstrata Interface
Propósito Compartilhar código e estado em uma hierarquia. Definir um contrato de comportamento (o que a classe faz).
Relação "É-UM" (IS-A). `ProfessorEfetivo` é um `Professor`. "PODE-FAZER" (CAN-DO). `Avaliacao` pode ser `Publicavel`.
Relação "É-UM" (IS-A). `ProfessorEfetivo` é um `Professor`. "PODE-FAZER" (CAN-DO). `Avaliacao` pode ser `Publicavel`.
Implementação Pode ter métodos concretos (com código) e abstratos. Tradicionalmente, apenas assinaturas. A partir do Java 8, pode ter métodos `default`.
Atributos Pode ter atributos de instância. Apenas constantes (`public static final`).
Herança Múltipla Não. Uma classe só pode herdar de UMA classe abstrata. Sim. Uma classe pode implementar VÁRIAS interfaces.

Polimorfismo: Sobrescrita vs. Sobrecarga (Não confunda!)

Critério Sobrescrita (Overriding) Sobrecarga (Overloading)
O que é? Redefinir um método herdado da superclasse. Ter vários métodos com o mesmo nome, mas com parâmetros diferentes.
Localização Entre duas classes (pai e filha). Dentro da mesma classe.
Assinatura Mesma assinatura (nome e parâmetros). Diferentes assinaturas (número ou tipo de parâmetros).
Quando decide? Tempo de Execução (Runtime). Tempo de Compilação (Compile-time).
Anotação Java `@Override` (altamente recomendado). Não se aplica.
Exemplo `class Carro extends Veiculo { void mover() {... } }` `class Calculadora { int somar(int a, int b); double somar(double a, double b); }`

3. Padrões de Projeto (Design Patterns) GoF Essenciais

Categoria Padrão Intenção (Resumo para Memorização)
Criacionais Singleton Garante uma única instância global para uma classe. (Ex: Conexão com BD).
Factory Method Deixa as subclasses decidirem qual objeto criar. (Ex: Fábrica de `Veiculo` cria `Carro` ou `Moto`).
Estruturais Adapter Adapta uma interface incompatível para outra que o cliente espera. (Ex: "Tomada universal").
Decorator Adiciona funcionalidades a um objeto dinamicamente, sem usar herança. (Ex: Adicionar criptografia a um fluxo de dados).
Comportamentais Strategy Permite que um algoritmo (estratégia) seja selecionado em tempo de execução. (Ex: Diferentes formas de calcular frete).
Observer Notifica múltiplos objetos (observadores) sobre mudanças em um objeto (sujeito). (Ex: Notificar usuários quando um produto volta ao estoque).

4. Foco na Banca: Instituto AOCP - Estratégia de Prova

Perfil da Banca:

  • Objetiva e Direta: Enunciados curtos, sem "pegadinhas" complexas.[10]
  • Dificuldade Média: Foco em conceitos e aplicação direta, não em problemas teóricos profundos.[11, 10]
  • Conceitual: Valoriza a definição correta dos pilares da POO e a identificação em trechos de código.[3, 12, 13]

Formato da Questão:

  • Múltipla escolha (geralmente 4 ou 5 alternativas).[14]
  • Questões de "assinale a alternativa que define corretamente..." são comuns.
  • Questões com pequenos trechos de código para analisar a saída ou identificar o conceito aplicado.

Estratégia de Resolução:

  1. Identifique a Palavra-Chave: No enunciado, procure por termos como "reutilização de código" (Herança), "ocultar detalhes" (Encapsulamento), "muitas formas" (Polimorfismo), "relação É-UM vs. TEM-UM" (Herança vs. Composição).
  2. Elimine Alternativas: Rapidamente descarte as alternativas que definem outros conceitos.
  3. Atenção às Linguagens: Lembre-se de particularidades. Exemplo: PHP não suporta sobrecarga de métodos tradicionalmente como Java, o que pode gerar um erro fatal.[15]

5. Pontos de Atenção e "Pegadinhas" Comuns

  • Pai p = new Filho();: Isso é Polimorfismo (upcasting). A variável `p` é do tipo `Pai`, mas o objeto é do tipo `Filho`. Métodos chamados por `p` executarão a versão da classe `Filho` (se sobrescritos). Contudo, `p` só pode chamar métodos que existem na classe `Pai`.
  • @Override em Java: É uma anotação para o compilador. Se você acha que está sobrescrevendo, mas errou a assinatura do método, o compilador avisará. Sem ela, você pode acidentalmente criar uma sobrecarga.
  • Herança vs. Composição na Prática: Se o problema pede flexibilidade para "trocar o comportamento em tempo de execução", a resposta quase sempre será Composição (usando o padrão Strategy, por exemplo).
  • Abstração não é "capturar tudo": É o oposto. É selecionar o essencial e ignorar o resto.[3]
