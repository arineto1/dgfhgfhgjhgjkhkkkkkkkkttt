Resumo para Concurso: Complexidade de Algoritmos

Página 1: Fundamentos, Notações e Regras de Cálculo

1. Tempo vs. Espaço

  • Complexidade de Tempo (T(n)): Mede o número de operações relevantes (ex: comparações, trocas) em função do tamanho da entrada ($n$). Não mede segundos.[1]
  • Complexidade de Espaço (S(n)): Mede o espaço auxiliar (memória extra) que um algoritmo precisa alocar em função do tamanho da entrada ($n$).[1]
    • Espaço O(1) (Constante): Ideal. O uso de memória não depende de $n$. Ex: Alocar uma única variável auxiliar (soma = 0).
    • Espaço O(n) (Linear): O uso de memória cresce com $n$. Ex: Alocar um novo vetor do mesmo tamanho que a entrada.
    • Trade-off (Exemplo): O Mergesort é rápido (O(n log n) tempo) mas gasta O(n) de espaço. O Quicksort (médio) tem o mesmo tempo, mas gasta apenas O(log n) de espaço (pilha de recursão).[2]

2. O Léxico da Análise Assintótica (O, $\Omega$, $\Theta$)

Isto é crucial para bancas como a AOCP, que cobram a diferença entre os casos.

Notação Apelido Conceito Foco de Concurso
$O$ (Big O) "Teto" Limite Superior. O algoritmo nunca será mais lento que isso.[3, 4] PIOR CASO (A garantia do pior cenário).[5, 6]
$\Omega$ (Big Omega) "Piso" Limite Inferior. O algoritmo nunca será mais rápido que isso.[3, 4] MELHOR CASO (O esforço mínimo obrigatório).
$\Theta$ (Big Theta) "Justo" Limite Exato ("Sanduíche"). Só existe se $O$ e $\Omega$ forem iguais.[3, 4] CASO MÉDIO ou Algoritmos de Custo Fixo (Ex: Mergesort).[2]

Hierarquia: Se um algoritmo é $\Theta(n)$, ele automaticamente também é $O(n)$ e $\Omega(n)$.

3. Regras Práticas para Calcular Complexidade

Ao olhar para um código na prova, aplique estas regras:

  • Regra 1: O(1) - Operações Constantes
    Atribuição (x = 10), Aritmética (a + b), Acesso a índice de vetor (vetor[i]).[7, 8]
  • Regra 2: Regra da Soma (Foco no Dominante)
    Se um algoritmo faz um passo $O(n)$ seguido de um passo $O(n^2)$, a complexidade total é a soma: $O(n) + O(n^2)$.
    Na análise, ignore termos de ordem inferior e constantes:
    • $O(n + n^2) \rightarrow O(n^2)$
    • $O(5n^2 + 100n + 5000) \rightarrow O(n^2)$ [9]
  • Regra 3: Regra do Produto (Loops Aninhados)
    Se um passo é executado dentro de outro, as complexidades se multiplicam.[10, 11, 12]
    Exemplo O(n^2): 
    para i de 1 até N:     // Executa N vezes
    para j de 1 até N: // Executa N vezes (para CADA i)
        operacao_O(1)
// Total: N * N = O(N^2)
    Pegadinha de Concurso (Loop Triangular): 
    para i de 1 até N:
    para j de 1 até i: // Depende de i
        operacao_O(1)
    • A soma das operações é $1 + 2 + 3 +... + N = \frac{N(N+1)}{2}$.[12]
    • Isso é $\frac{N^2}{2} + \frac{N}{2}$. Aplicando a Regra 2 (Dominante), a complexidade ainda é $O(N^2)$.
  • Regra 4: O(log n) - Loops Logarítmicos
    Ocorre quando o loop corta o problema por um fator constante a cada iteração (ex: divide por 2, multiplica por 2 até $n$).[13, 14] 
    i = 1
enquanto (i < n):
    i = i * 2  // i -> 1, 2, 4, 8, 16... (crescimento logarítmico)

5. Estudo de Caso Obrigatório: Quicksort (O "Duas Caras")

O Quicksort é o favorito das bancas porque seus casos de performance variam drasticamente.

  • Pior Caso: $O(n^2)$ [22]
    • Quando Ocorre? Quando o pivô escolhido é consistentemente ruim (o menor ou o maior elemento).
    • Gatilho (Memorize): Acontece se o vetor de entrada já está ordenado (ou em ordem inversa) e o pivô é sempre o primeiro/último elemento.[22] A partição fica desbalanceada (N-1 e 1).
  • Melhor/Médio Caso: $\Theta(n \log n)$ [19, 22]
    • Quando Ocorre? Quando o pivô é "bom" (mediana ou aleatório) e divide o vetor em metades balanceadas ($N/2$ e $N/2$).
