Resumo de Elite: UML

Foco: Memorização e "Pegadinhas" da Banca Instituto AOCP (IFPB)

Resumo em Áudio:

Este resumo condensa a especificação UML 2.5 em pontos-chave de memorização. O Instituto AOCP cobra sintaxe visual exata e definições literais.

1. Taxonomia: Os 14 Diagramas (7 + 7)

A primeira questão clássica é classificar o diagrama. Use a regra de exclusão: memorizar os estruturais (estáticos). Se não for estrutural, é comportamental (dinâmico).

ESTRUTURAIS
(Estáticos - "Aquilo que É")		 COMPORTAMENTAIS
(Dinâmicos - "Aquilo que FAZ")
  • Classes (O mais cobrado)
  • Objetos (Retrato instantâneo)
  • Componentes (Módulos)
  • Implantação (Hardware/Infra)
  • Pacotes (Organização)
  • Estrutura Composta
  • Perfil
  • Casos de Uso (Funcional)
  • Atividades (Fluxo)
  • Máquina de Estados
  • Sequência (Interação)
  • Comunicação (Antigo Colaboração)
  • Tempo
  • Visão Geral de Interação
Mnemônico AOCP: Se o diagrama mostra tempo, mensagem, evento ou processo, ele é Comportamental.
Se mostra arquivos, classes ou nós, é Estrutural.

2. Diagrama de Classes: O Rei das Questões

Foco total nos relacionamentos. A banca troca os diamantes para confundir.

Tabela da Verdade dos Relacionamentos

Relação Símbolo Palavras-Chave da Banca
Associação ─── "Usa", "Conhece", "Vinculado a"
Agregação ◇ "Parte-Todo", "Fraco", "Independente". O filho existe sem o pai.
Composição ◆ "Forte", "Dependente", "Destruição em cascata". Filho morre com o pai.
Generalização △ "É um", "Herança", "Especialização"
Realização △ - - "Implementa", "Contrato" (Interface)
Dependência < - - "Usa temporariamente", "Parâmetro"
Macete de Memorização:
  • Composição = Com vida (Diamante Preenchido). Se o pai morre, o filho morre.
  • Agregação = Aberto (Diamante Vazio). A parte é livre.
Multiplicidade (Cardinalidade):
0..1: Opcional (Zero ou Um)
1: Obrigatório (Exatamente um)
0..*: Zero ou muitos
1..*: Pelo menos um

3. Casos de Uso: Include vs. Extend

O erro número 1 em provas é inverter a direção da seta ou a definição.

Tipo Obrigatório? Direção da Seta (Tracejada)
<<include>> SIM (Indispensável) Base ---> Incluído
(Aponta para quem é usado)
<<extend>> NÃO (Opcional/Extra) Extensão ---> Base
(Aponta para quem é estendido)
Atenção: O Extend funciona como um Insert. A seta sai da extensão (o pedaço novo) e aponta para onde ela vai se encaixar (o caso de uso base).

4. Diagrama de Sequência: Fragmentos

A AOCP pergunta o significado dos operadores no canto superior esquerdo.

  • alt Alternatives (IF / ELSE): Apenas UM caminho executa.
  • opt Option (IF sem else): Executa ou não.
  • loop Repetição (FOR / WHILE).
  • par Parallel: Threads executando ao mesmo tempo.
  • break Interrompe o fluxo (Throw Exception).

5. Diagrama de Atividades

Diferença visual crítica entre Paralelismo e Decisão.

Losango (◇)

Decisão (if) ou Fusão (Merge).
Apenas um fluxo passa.

Barra Espessa (Preta)

Fork: Entra 1, saem vários (Paralelismo).
Join: Entram vários, sai 1 (Sincroniza).

Pegadinha: Fork deve fechar com Join. Se fechar com Losango (Merge), gera erro lógico.

6. Componentes e Implantação

  • Bola e Soquete:
    • Bola (Lollipop): Interface Fornecida (Eu faço!).
    • Soquete (Meia-lua): Interface Requerida (Eu preciso!).
  • Nó (Cubo 3D): Hardware ou Ambiente de Execução (Docker).
  • Artefato: Arquivo físico (.jar, .war). Vive dentro do nó.
