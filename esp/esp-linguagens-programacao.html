Python 3: Pontos-Chave para Memorização

Filosofia e Sintaxe

  • Indentação é Obrigatória: Usa 4 espaços para definir blocos de código (if, for, def, etc.). Indentação errada causa IndentationError. Não usa chaves {}.
  • Zen of Python: Preza por código simples, legível e explícito.

Tipagem: Dinâmica e Forte

  • A tipagem dinâmica não impede a tipagem forte:
  • x = 10 # x é inteiro
  • y = "2" # y é string
  • print(x + y) # GERA ERRO! Python é Forte, não permite somar Int com String implicitamente.

Estruturas de Dados Nativas (Tópico Clássico de Prova)

  • list: Coleção ordenada e MUTÁVEL. Ex: [1, "a", 3.0]. Métodos: .append(), .pop(), .remove().
  • tuple: Coleção ordenada e IMUTÁVEL. Uma vez criada, não pode ser alterada. Ex: (1, "a", 3.0). Tentar alterar um item causa TypeError.
  • dict: Coleção de pares chave:valor. Chaves devem ser imutáveis (números, strings, tuplas). Ex: {'nome': 'João', 'idade': 30}.
  • set: Coleção de elementos ÚNICOS e não ordenada. Útil para remover duplicatas. Ex: {1, 2, 3}.

Tópicos Avançados Frequentes

  • List Comprehensions: Forma "pythônica" e concisa de criar listas. Ex: quadrados = [x**2 for x in range(10)].
  • Tratamento de Exceções: Estrutura try...except...else...finally.
    • Ponto de atenção: O bloco finally SEMPRE é executado, ocorrendo ou não uma exceção.

JavaScript (ECMAScript): Pontos-Chave para Memorização

Declaração de Variáveis e Escopo (A "Pegadinha" Mais Comum)

Declaração Escopo Hoisting Reatribuição/Redeclaração
var Escopo de FUNÇÃO Sim (inicializa como undefined) Pode ser reatribuída E redeclarada.
let Escopo de BLOCO {} Sim (mas não inicializa) Pode ser reatribuída, mas não redeclarada.
const Escopo de BLOCO {} Sim (mas não inicializa) NÃO PODE SER REATRIBUÍDA.

Exemplo de Bloco (Tópico de Prova)

var x = 1;
let y = 1;
if (true) {
  var x = 2; // Redeclara a variável x fora do bloco (Escopo de Função)
  let y = 2; // Cria uma nova variável y, local ao bloco (Escopo de Bloco)
  console.log(x); // Saída: 2
  console.log(y); // Saída: 2
}
console.log(x); // Saída: 2 (valor global foi alterado)
console.log(y); // Saída: 1 (o y do bloco foi destruído, valor global mantido)

O Comportamento do this (Complexo e Essencial)

  • Método de Objeto: obj.metodo() -> this é o obj.
  • Função Simples: minhaFuncao() -> this é o objeto global (ou undefined em "use strict";).
  • Construtor: new Funcao() -> this é o novo objeto criado.
  • Arrow Function (=>): NÃO POSSUI this PRÓPRIO. Herda o this do escopo onde foi definida.

Assincronismo e o Event Loop

  • Modelo: JavaScript é single-threaded. O Event Loop gerencia operações assíncronas sem bloquear a thread principal.
  • Evolução (Entender a ordem):
    1. Callbacks: Funções passadas como argumento (pode levar ao "Callback Hell").
    2. Promises (ES6): Objeto que representa um valor futuro. Estados: pending, fulfilled, rejected.
    3. async/await (ES8): "Açúcar sintático" sobre Promises. Permite escrever código assíncrono com aparência síncrona.

Página 2: C/C++ e Java

C/C++: Pontos-Chave para Memorização

Gerenciamento de Memória (O Tópico Mais Crítico)

  • Manual: O programador é 100% responsável por alocar e liberar memória. Erros comuns: memory leak (vazamento) e dangling pointers (ponteiros pendentes).
  • Ponteiros: Variáveis que armazenam endereços de memória.
    • &variavel: Operador "endereço de". Retorna o endereço da variável.
    • *ponteiro: Operador de "desreferenciação". Acessa o valor no endereço.
  • Alocação Dinâmica (Diferença Fundamental - Questão de Prova):
    • malloc() / free() (Estilo C): Alocam memória bruta. NÃO CHAMAM CONSTRUTOR OU DESTRUTOR.
    • new / delete (Estilo C++): Alocam memória E CHAMAM O CONSTRUTOR. delete CHAMA O DESTRUTOR.

Standard Template Library (STL)

  • std::vector: Array dinâmico.
  • std::list: Lista duplamente encadeada.
  • std::map: Dicionário (chave-valor) ordenado.

Java: Pontos-Chave para Memorização

A Arquitetura da JVM (Java Virtual Machine)

  • Portabilidade: O lema "Write Once, Run Anywhere" (Escreva uma vez, rode em qualquer lugar) é possível porque o código Java é compilado para bytecode, que é executado pela JVM.
  • Estrutura de Memória (Questão Clássica de Concurso):
    • Heap: Área de memória onde TODOS OS OBJETOS e arrays são alocados. Gerenciada pelo Garbage Collector (GC).
    • Stack (Pilha): Cada thread tem sua própria pilha. Armazena variáveis locais (tipos primitivos) e REFERÊNCIAS para os objetos que estão no Heap.

O Ponto de Entrada: public static void main(String args)

  • Cada palavra-chave tem um motivo para a JVM iniciar o programa:
    • public: Para que a JVM (externa) possa acessá-lo.
    • static: Para que a JVM possa chamar o método sem precisar criar um objeto da classe.
    • void: O método não retorna nenhum valor para a JVM.
    • main: Nome padrão que a JVM procura.

Java Collections Framework (JCF)

  • List: Coleção ordenada. (ArrayList, LinkedList).
  • Set: Coleção que não permite elementos duplicados. (HashSet).
  • Map: Coleção de pares chave-valor. (HashMap).
