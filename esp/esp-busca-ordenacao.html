Página 1: O Essencial de Busca e o Grande Quadro da Ordenação

1. Algoritmos de Busca: Foco em Complexidade e Pré-requisitos

A chave para acertar questões de busca é identificar se a coleção de dados está ordenada.

Característica Busca Linear (Sequencial) Busca Binária
Pré-requisito Nenhum (Funciona em dados desordenados) Dados devem estar ordenados
Estratégia Força Bruta (um a um) Divisão e Conquista (corta pela metade)
Pior Caso (Tempo) O(n) (Lento para grandes listas) O(log n) (Extremamente rápido)
Espaço Auxiliar O(1) (in-place) O(1) (iterativa) / O(log n) (recursiva)
Quando Usar? Listas pequenas ou desordenadas. Listas grandes e ordenadas, com buscas frequentes.

Mantra de Memorização:

  • Busca Linear: Dados desordenados? Força bruta. Complexidade O(n).
  • Busca Binária: Dados ordenados? Dividir para conquistar. Complexidade O(log n).

2. O Grande Quadro: Tabela Comparativa Definitiva dos Algoritmos de Ordenação

Esta tabela é a ferramenta mais importante para memorização. As colunas mais cobradas são Pior Caso (Tempo), Espaço Auxiliar e Estabilidade.

Algoritmo Melhor Caso Médio Caso Pior Caso Espaço Estabilidade In-Place?
Bubble Sort O(n) (otimizado) O(n2) O(n2) O(1) Sim Sim
Selection Sort O(n2) O(n2) O(n2) O(1) Não Sim
Insertion Sort O(n) O(n2) O(n2) O(1) Sim Sim
Merge Sort O(n log n) O(n log n) O(n log n) O(n) Sim Não
Quick Sort O(n log n) O(n log n) O(n2) O(log n) Não Sim
Heap Sort O(n log n) O(n log n) O(n log n) O(1) Não Sim

Dicas de Memorização:

  • Quadráticos (O(n2)): Bubble, Selection, Insertion. São simples, mas lentos para grandes volumes.
  • Log-Lineares (O(n log n)): Merge, Quick, Heap. São os mais eficientes.
  • Estáveis: Lembre-se de BIM: Bubble, Insertion, Merge. A estabilidade preserva a ordem original de elementos iguais.
  • Espaço: A grande desvantagem do Merge Sort é o espaço auxiliar O(n). Heap Sort e os quadráticos usam espaço O(1). Quick Sort usa O(log n) na pilha de recursão.

Página 2: Fichas de Algoritmos e Cenários de Prova

3. Fichas de Memorização Rápida por Algoritmo

Bubble Sort

  • Ideia Central: "Flutuação" do maior elemento para o final a cada passagem.
  • Ponto-Chave: Melhor caso é O(n) se otimizado com uma flag para detectar ausência de trocas.
  • Estabilidade: Sim.

Selection Sort

  • Ideia Central: Encontrar o menor elemento e trocá-lo para a posição correta.
  • Ponto-Chave: Realiza o mínimo de trocas (n-1), mas a complexidade é sempre O(n2).
  • Estabilidade: Não.

Insertion Sort

  • Ideia Central: Como organizar cartas de baralho, inserindo cada elemento na parte já ordenada.
  • Ponto-Chave: Melhor caso é O(n). Extremamente eficiente para dados quase ordenados.
  • Estabilidade: Sim.

Merge Sort

  • Ideia Central: Dividir para conquistar. Divide o vetor ao meio recursivamente e depois intercala (merge) as partes ordenadas.
  • Ponto-Chave: Desempenho garantido de O(n log n) em todos os casos, mas precisa de espaço auxiliar O(n).
  • Estabilidade: Sim.

Quick Sort

  • Ideia Central: Dividir para conquistar usando um pivô. Particiona o vetor em menores e maiores que o pivô.
  • Ponto-Chave: Geralmente o mais rápido na prática, mas tem um pior caso de O(n2) se o pivô for mal escolhido (ex: vetor já ordenado).
  • Estabilidade: Não.

Heap Sort

  • Ideia Central: Usa a estrutura de dados Max-Heap para colocar o maior elemento sempre na raiz e trocá-lo para o final.
  • Ponto-Chave: Desempenho garantido de O(n log n) e é in-place (espaço O(1)).
  • Estabilidade: Não.

4. Cenários de Prova: Como Escolher o Algoritmo Certo

Use este guia de decisão rápida para questões de múltipla escolha.

  • SE a memória é crítica (pouco espaço) E o desempenho no pior caso deve ser O(n log n)...
    ENTÃO a resposta é Heap Sort. (É in-place e garantido).
  • SE os dados estão quase sempre ordenados e precisam de reordenações frequentes...
    ENTÃO a resposta é Insertion Sort. (Aproveita o melhor caso O(n)).
  • SE a estabilidade é um requisito obrigatório (preservar a ordem original de itens iguais)...
    ENTÃO a resposta é Merge Sort. (É o único estável com desempenho O(n log n) garantido).
  • SE a velocidade média é a maior prioridade para dados genéricos e grandes...
    ENTÃO a resposta é Quick Sort. (Na prática, costuma ser o mais rápido, apesar do risco do pior caso).
