1. Análise de Complexidade (Big O) - A Métrica Essencial

A Notação Big O mede como o tempo de execução ou memória de um algoritmo cresce com o aumento da entrada (n). É a principal ferramenta para comparar a eficiência de estruturas de dados.

Notação Nome Descrição (Como o tempo cresce) Exemplo Clássico
O(1) Constante Não cresce. Tempo fixo. Acessar um elemento de um vetor pelo índice.
O(log n) Logarítmica Cresce muito slowly. "Divide e conquista". Busca binária em um vetor ordenado.
O(n) Linear Cresce na mesma proporção que a entrada. Percorrer uma lista para encontrar um valor.
O(n log n) Log-Linear Padrão de algoritmos de ordenação eficientes. Merge Sort, Quick Sort (caso médio).
O(n²) Quadrática Cresce com o quadrado da entrada. Laços aninhados (Bubble Sort).

2. Estruturas Lineares Sequenciais: Vetor vs. Lista Ligada

A escolha fundamental entre alocação contígua (Vetor) e dinâmica (Lista Ligada).

Característica Vetor (Array) Lista Ligada (Linked List)
Memória Contígua (bloco único) Não Contígua (nós com ponteiros)
Acesso (por índice) O(1) (Excelente) O(n) (Ruim)
Busca (por valor) O(n) O(n)
Inserção/Remoção O(n) (Lento, exige deslocamento) O(1) (Rápido, ajusta ponteiros)
Uso de Memória Apenas dados Dados + overhead de ponteiros
Performance de Cache Alta (Cache-friendly) Baixa (Cache-unfriendly)
Cenário Ideal Acesso frequente por índice; tamanho estável. Inserções/remoções frequentes; tamanho dinâmico.

3. Estruturas Lineares com Política de Acesso

3.1. Pilha (Stack)

  • Princípio: LIFO (Last-In, First-Out) - O último a entrar é o primeiro a sair.
  • Analogia: Pilha de pratos.
  • Operações: push (empilhar) e pop (desempilhar).
  • Complexidade: O(1) para push e pop.
  • Aplicações Chave:
    • Pilha de Execução de Funções (incluindo recursão).
    • Função "Desfazer" (Ctrl+Z) em editores.
    • Botão "Voltar" em navegadores web.
    • Análise de expressões (ex: balanceamento de parênteses).

3.2. Fila (Queue)

  • Princípio: FIFO (First-In, First-Out) - O primeiro a entrar é o primeiro a sair.
  • Analogia: Fila de banco.
  • Operações: enqueue (enfileirar) e dequeue (desenfileirar).
  • Complexidade: O(1) para enqueue e dequeue.
  • Aplicações Chave:
    • Escalonamento de processos em Sistemas Operacionais.
    • Filas de impressão.
    • Buffers de dados em rede.
    • Algoritmo de Busca em Largura (BFS) em grafos.

4. Estruturas Hierárquicas: Árvores

4.1. Árvore Binária de Busca (BST)

  • Propriedade Fundamental: Para qualquer nó N, subárvore_esquerda < N < subárvore_direita.
  • Complexidade Média: O(log n) para busca, inserção e remoção.
  • Pior Caso (Ponto Crítico para Concursos): Se os dados são inseridos em ordem (crescente/decrescente), a árvore degenera em uma lista ligada. A altura se torna n, e a complexidade de todas as operações degrada para O(n).

4.2. Árvores Balanceadas: AVL vs. Rubro-Negra

Resolvem o pior caso da BST, garantindo altura O(log n) através de rotações.

Característica Árvore AVL Árvore Rubro-Negra (Red-Black)
Balanceamento Rígido (diferença de altura ≤ 1) Flexível (baseado em cores e altura negra)
Altura Menor (mais balanceada) Ligeiramente maior que AVL
Busca Mais rápida (ideal para read-heavy) Rápida (O(log n))
Inserção/Remoção Mais lenta (mais rotações) Mais rápida (ideal para write-heavy)
Uso Comum Dicionários de dados estáticos std::map (C++), TreeMap (Java)

5. Estruturas de Acesso Otimizado

5.1. Tabela Hash (Hash Table)

  • Objetivo: Mapear chaves a valores com acesso, inserção e remoção em tempo médio O(1).
  • Mecanismo: Função de Hash converte uma chave em um índice de um vetor.
  • Colisão: Ocorre quando duas chaves diferentes geram o mesmo índice.
Tratamento de Colisão Descrição Vantagem Principal Desvantagem Principal
Encadeamento Separado Cada índice aponta para uma lista ligada com os elementos que colidiram. Simples de implementar; fator de carga pode ser > 1. Overhead de memória dos ponteiros; pior caso O(n).
Endereçamento Aberto Procura por uma posição livre na própria tabela (sondagem). Melhor performance de cache; sem overhead. Remoção complexa (requer tombstones).

5.2. Heap (Fila de Prioridade)

  • Objetivo: Implementar uma Fila de Prioridade de forma eficiente.
  • Propriedade do Heap:
    • Max-Heap: O pai é sempre maior ou igual aos filhos (raiz é o máximo).
    • Min-Heap: O pai é sempre menor ou igual aos filhos (raiz é o mínimo).
  • Estrutura: Árvore binária quase completa, garantindo altura O(log n).
  • Complexidade:
    • Acessar o máximo/mínimo: O(1).
    • Inserir ou Remover o máximo/mínimo: O(log n).
  • Aplicações: Algoritmo de Dijkstra, Heapsort, escalonadores de SO.
